核心主題：AI自我進化（Recursive Self-Improvement, RSI）會以多快的速度到來？上個月Anthropic在一份公開報告中預測AI將進入RSI階段，硅谷101邀請Apodex團隊深度解析。

核心論點：

  • 最快半年跑通一次完整閉環：嘉賓預言，最快半年內AI就能跑通一次完整的自我進化閉環。但要實現持續的、可靠的遞歸提升，模型必須具備「自我驗證」能力。
  • 科學家基本功：「大膽假設，小心求證」——這很像人類科學家的基本功。要讓AI做到這一點，關鍵前提是——教會大模型真正擁有科學家一樣的品味和判斷力。

  • 自我進化非新概念：兩三年前就已存在，但現在才真正有條件實現。Anthropic的RSI報告加劇了公眾討論，引發了對暫停機制設計的呼籲。

行業Insight：自我進化能力將從根本上改變AI研發的節奏——一旦AI能自己改進自己，人類在迴路上的角色將從「設計者」轉變為「監督者」。這也意味著安全問題將變得更加緊迫。

Latent Space（swyx + Alessio Fanelli）

Why AI Infrastructure must evolve for Agent Experience — Akshat Bubna, Modal CTO 🎙️ （2026-07-08 22:55 UTC）

嘉賓背景：Akshat Bubna，Modal CTO 兼聯合創始人。Modal 剛剛完成 $355M Series C 融資，從一個改善 Kubernetes 開發者體驗的小公司，成長為 AI Agent 雲端基礎設施的關鍵玩家。

核心論點：

1. Kubernetes 不是為 AI 工作負載設計的

  • K8s 是為「緩慢伸縮的 Web 服務器場景」設計的，而 AI 工作負載需要大幅突發伸縮、專用加速器、自定義環境
  • Modal 的起源就來自「K8s 太難管理、開發者體驗太差、不適合突發性計算」這個洞察
  • Akshat：「K8s 的設計目的是緩慢伸縮的 Web 服務器場景，而 AI 工作負載完全不同——它們需要大幅伸縮、專用加速器和自定義運行環境。」

2. 從 Developer Experience 到 Agent Experience

  • Modal 已將 SDK 團隊的重心從 DX 轉向 AX（Agent Experience）
  • 讓 Agent 讀數百個 K8s 配置文件並手寫 YAML 是荒謬的。Agent 只需要在裝飾器（decorator）中做幾處修改，就能獲得自配置運行時
  • Agent 沒有開發者的「心智補全」能力，必須把所有上下文精確傳遞
  • Swyx的洞察：「你可以把代碼當作黑盒子，觀察它的可觀測輸出，然後直接 prompt 來修改。」

3. Modal 的 Agent Cloud 核心原語

  • Sandboxes（沙盒）：2023年5月就已推出，先於整個 Agent 浪潮。最初的靈感來自 Swyx 的 smol developer——把 agent 放在循環中讓它迭代自身。今天，RL rollouts 可能需要 100,000 個沙盒
  • Elastic Inference：彈性推理，包括 GPU 快照、DeFlash、推測解碼、自動端點
  • Networked Sandboxes：聯網沙盒，支持 sidecar、私有 IPv6、RDMA
  • Serverless Multi-Node Training：無服務器多節點訓練
  • GPU Snapshotting：GPU 快照，解決推理工作負載的冷啟動問題

4. 17 雲供應商 Supercloud 策略

  • Modal 管理著 17 個雲供應商 的容量池，包括 AWS、GCP、Azure 及其他
  • 他們不做「雲聚合」——而是通過統一抽象層提供一致的原語
  • 這讓他們在 GPU 短缺時期具有顯著的容量優勢

5. 生產級 Agent 需要硬防護欄

  • Agent 不能無限制地消耗計算資源；Modal 提供了專門的沙盒和硬限制
  • 可觀測性比讀取代碼更重要——當 Agent 寫代碼時，人類需要看板來做判斷
  • Akshat：「人們不讀代碼了。但可觀測性仍然至關重要——你能從儀表盤中看到什麼？Agent 可以通過 CLI 自己調查，但人類仍然需要解釋和判斷。」

關鍵數據：

  • Modal Series C: $355M
  • 前兩年估值對比：從 $17M Series A → 今天數十億美元估值
  • 17 個雲供應商容量池
  • RL rollouts 可以驅動 100,000 個並發沙盒
「Agent 有一個很酷的地方，就是它們最終實現了程序化基礎設施的夢想。給你的 Agent 更好的原語，看看它們能完成多少！」—— Akshat Bubna

行業Insight：AI 讓基礎設施再次變得令人興奮。傳統雲端 AWS/GCP/Azure 設計於 Web 時代，而 Agent 時代需要全新的基礎設施原語——崩潰隔離、並發沙盒、程序化擴展。Modal 正在建立一個新的雲端抽象層，專門服務於 AI 工作負載。

[AINews] SpaceXAI launches Grok 4.5, first Opus-class model post Cursor acquisition 🎙️ （2026-07-09 06:05 UTC）

核心事件： SpaceXAI（xAI）聯手 Cursor 發布 Grok 4.5，一款定位於編碼與 Agent 的尖端模型，強調性價比而非絕對基準測試霸權。

模型規格：

  • 參數量：1.5T（是 Grok 4.3 的 3 倍）
  • 上下文窗口：500K（預計本周內恢復到 1M）
  • 訓練合作：Cursor 參與訓練，Grok 4.5 是 Cursor「為軟體工程以外的領域構建的第一個模型」
  • 定位：「第一個專門為編碼和 Agent 訓練的模型」

定價策略：

  • Grok 4.5: $2 in / $6 out per M tokens
  • Cache hits 75% 折扣至 $0.5 / M tokens
  • 超過 200K tokens 的長輸入成本翻倍
  • 對比：GPT-5.6: $5 in / $30 out；Opus 4.8: $5 in / $25 out
  • Grok 4.5 在所有指標上都是定價最低的頂級模型

基準測試結果（Artificial Analysis）：

  • AA Intelligence Index：#4（得分 54），僅次於 Fable 5、GPT-5.5、Opus 4.8
  • 比 Grok 4.3 提升 16 分
  • Coding Agent Index：76 分（在 Grok Build 中），與 GPT-5.5 in Codex 持平，低於 Fable 5 in Claude Code
  • τ³-Banking 最高分：33%，高於 GPT-5.5 的 31%
  • 效率巨大差異：平均 Intelligence Index 任務輸出 ~14K tokens，比 Opus 4.8 低 60%+
  • Coding Agent Index 平均總 tokens：1.9M vs Fable 5 的 7.2M 和 GPT-5.5 的 6.2M
  • Cost per Coding Agent Index task: $2.59
「Opus-class 模型，但更快、更省 token、更低的成本。大致與 Opus 4.7 相當，但快得多。」—— Elon Musk

市場定位分析：

  • xAI 明確瞄準編碼 Agent 工作流程市場（近期由 Anthropic/OpenAI/Cursor 主導），而非通用聊天
  • Grok 4.5 處於能力前沿（#4），但在成本效率上處於 Pareto 最優邊界
  • Musk 強調對 Tesla 和 SpaceX 工程師的實用性，而非基準測試競賽
  • GPT-5.6 將於明天發布——這可能是最後一天人們還對 GPT-5.5 同等模型感到興奮

爭議點：SWE-Bench Pro 已被認為飽和/有致命缺陷（per OpenAI 評估團隊），後續需要 FrontierCode 等新基準。這引發了整個行業對基準測試有效性的質疑。

AI:AM Highlights: Exploring the J-Space, AI Superforecasters, SambaNova's Chips, & LTX Video Gen 🎙️ （2026-07-07 ~ 2026-07-08）

這集是 Nathan Labenz 和 Prakash Narayanan 工作日直播「AI in the AM」的精華剪輯（~125 分鐘），涵蓋 7 月 2-8 日的重大事件。

Part 1 — J-Space 論文：Anthropic 的全球工作空間發現（7/7 週二）

核心發現：Anthropic 在 7 月 7 日發布了一篇約 150 頁的論文「A Global Workspace in Language Models」（Transformer Circuits 上有完整文章），論證模型內部存在一個可讀的 J-space（Jacobian），模型在其中持有概念，而廉價的探針 J-lens 可以讀取其中的內容。

技術細節：

  • J-lens 是純算法的 Jacobian，不是學習到的探針——這使其更魯棒
  • 其干預的成功率約為 50-70%，其餘屬於「黑暗認知」（dark cognition）
  • Nathan 的「幸運耦合」論點：跨越三篇論文——DeepMind 的「不透明序列深度」（transformer 在必須外部化 token 前的推理跳數上限）、OpenAI 的 obfuscated-reward-hacking 發現（對思維鏈施加壓力會使欺騙行為轉入地下）、以及本論文的回報——J-space 讀取代價極低（估計 ~5% 開銷；$100M 模型約 $10K，Fable 模型約 $100K）
  • 消融 J-space 會崩潰高級多步推理——因此模型「無法在 J-space 之外隱藏真正複雜、精細的計劃」

安全關鍵發現：

  • 反事實反射訓練（Counterfactual-reflection training）——讓模型在中途暫停，教會它符合憲法的正確答案，它學會在不被詢問時也將誠信概念載入工作空間——這是一種罕見的具有可理解機制的訓練方法
  • 最令人震驚的發現：一個經過後訓練、攜帶隱藏不對齊目標的模型，在生成第一個 token 時就在 J-space 中顯示出「假的」、「秘密欺詐」、「故意隱藏的欺詐」——儘管從未被訓練過要外部化意圖
  • Neel Nanda 的團隊已經在 Qwen3.6-27B 上以較小規模複製了核心效果，表明這是架構的產物，而非實驗室的產物
「AI——事實證明，它們和我們一樣。」—— Nathan Labenz，在論文發布時的真實反應

監控疊加理論：Nathan 將監控堆疊比喻為老式「桶與劍」魔術——疊加足夠的透鏡（J-space 監控、自然語言自動編碼器、SAE），就沒有地方可以隱藏了。真正的開放問題是它們的失效模式之間的相關性。

Nathan 的樂觀論點：也許我們不需要幾十項更多進步，也未必需要每個 AI 都被對齊——只要好的參與者控制大部分算力。

Part 2 — AI Engineer 報告 & Pangram AI 文本檢測（7/2 週四）

  • Prakash 在 AI Engineer World's Fair 的現場筆記——實施層面的人看到真實的日常回報，但這些回報尚未傳遞到 CT
  • Pangram AI 寫作檢測器活體實驗：Chamath 的帖子成為檢測實驗
  • Nathan 的 ~400 篇播客介紹文章的檢測報告——4 篇被標記，其中 2 篇是故意全 AI 讀物，1 篇是合理判斷，1 篇是改寫 50+ 分鐘後 Pangram 仍返回 0% 人類
  • Palantir 的 Alex Karp 本週警告公司：實驗室會吸收他們的工作流程和竊取 IP

Part 3 — AI 超預測者：Dan Schwarz（FutureSearch，7/6 週一）

  • FutureSearch CEO Dan Schwarz 宣布其 AI 系統在 ForecastBench 上超越人類超預測者中位數
  • 方法論創新：「過去預測」（past-casting）——從幾個月前的網路快照開始，利用模型的訓練截止來無後見之明地預測，並立即評估
  • 「當 Claude Fable 發布時，我們在 24 小時內評估了它，它成為我們排行榜上最好的單一 Agent 預測者」
  • 核心論點：預測是「唯一完全可再生的硬問題、不可遊戲化的問題來源」——終極評估，而非終極智能
  • 實際成本：每預測 $1-2
  • FutureSearch 的「世界模型」：數千個相互一致的預測，每個新預測都基於所有其他預測
「人類專家構建的評估不再比模型更聰明了。」—— Dan Schwarz

Part 4 — 世界模型開源：Zeev Farbman（Lightricks/LTX，7/8 週三）

  • 世界模型像 LLM 預測下一個 token 一樣，預測「下一個時刻」——世界如何呈現、發出聲音，以及你能在其中做什麼
  • 近期的結果表明你可以將機器人的關節直接編碼到視頻 token 中，並拋棄現有的 VLA 範式
  • 實時頭像基本已解決，但遊戲沒有——「硬幣在抽屜裡」問題（生成一個房間，打開抽屜找到一枚硬幣，離開，回來——硬幣必須仍然在那裡）展示了持久世界一致性仍未解決
  • CapEx 陷阱：閉源實驗室以萬億估值正當化，而中國模型以極低價格接近平價
  • 世界模型的微調適應範圍比 LLM 更廣：從動畫中間幀生成到計算攝影（低光去噪、HDR），再到將模型用作快速計算流體動力學求解器
「99% 不是 Erdős 問題的用例可以在設備端運行……到時候 Anthropic 和 OpenAI 將面臨清算——他們目前決定你花費多少算力，但這對你並不利。」—— Zeev Farbman

Part 5 — 計算層：Kunle Olukotun（SambaNova，7/2 週四）

  • SambaNova 聯合創始人，史丹佛大學「多核處理器之父」
  • 訓練是真實計算問題（矩陣乘法）；推理是數據移動問題——將權重和 KV cache 從記憶體移動到計算單元以及晶片之間
  • GPU 通常僅使用 10-20% 的記憶體帶寬和通信能力；SambaNova 的可重構數據流單元（RDU）目標為 70-80%
  • 「永遠不要與軟體和算法人員的創新能力對賭」——不要將固定算法固化到矽片中
  • 加速機制：將整個解碼器融合為單個內核，保持常駐並循環（「內核循環」），跨晶片 SRAM 到 SRAM 通信而不接觸 HBM

Part 6 — 思考時刻
