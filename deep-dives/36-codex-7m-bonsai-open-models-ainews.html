三、Bonsai 27B：手機級嘅 AGI 可能性

Daniel Han（@danielhanchen）宣布了 NVFP4 dynamic quants 支援 Gemma-4 家族、Qwen3.5-122B-A10B、GLM-4.7-Flash 等模型。Mia AI Lab 嘅 DGX Spark thread 仲展示咗實用 multi-node local deployment 配置——2× DGX Sparks 可以跑 1M-context DeepSeek v4 Flash 同 MiMo-V2.5。

四、即時多模態：MOSS-VL-Realtime 同 OmniAgent

呢日仲有兩個重要嘅多模態 release，方向唔同但互相補充。

MOSS-VL-Realtime（OpenMOSS）

11B · Apache 2.0 · 256K context

呢個模型嘅關鍵特性：可以一邊生成一邊繼續睇。傳統嘅 VLM 係「先睇曬再答」，MOSS-VL-Realtime 係持續感知——場景變化時可以中斷/修正回答，證據不足時保持沉默。

技術架構方面：cross-attention + XRoPE（統一時空定位）+ unified templates 覆蓋離線、串流同實時三種設定。

OmniAgent（基於 Qwen2.5-Omni-7B）

呢個更激進。OmniAgent 唔係被動 ingest frames，而係行 Observation–Thought–Action 循環——只請求佢需要嘅 frame 同 audio。

結果：

  • LVBench 得分 50.5 vs Qwen2.5-VL-72B 嘅 47.3
  • 只消耗 ~203 frames vs 768

訓練方法都好有意思：純 SFT 反而降低 performance，真正有效嘅係 58K agentic trajectories + 熵加權 RL（TAURA 框架）。

Andrew Carr 對此有一個重要嘅觀察：

「Motion is a fundamentally novel data type requiring dedicated collection, infra, and model treatment rather than being reduced to images-with-time.」

即係話：唔好將運動視為「有時間戳嘅圖片」，佢係全新嘅資料類型，需要專門嘅採集基建同模型設計。

五、評估方法論：WANDR、Agent Arena、SlopCode

評估（eval）領域呢日有幾個好重要嘅更新。

Perplexity 開源 WANDR

500 tasks · 170,495 條來源記錄

WANDR 係一個「wide-and-deep agentic research」benchmark——唔係用靜態答案庫評分，而係動態 re-fetch 引用頁面，驗證 claims against underlying evidence。呢個設計更貼近真實嘅網頁研究場景。

Arav Srinivas（Perplexity CEO）話呢個係 Perplexity Computer deep-and-wide research harness 嘅內部 benchmark。Denis Yarats 仲補充：WANDR 同時可以作為一個 RL environment，由 production traces 合成。

Agent Arena

Agent Arena 發表咗一個 important result：用 89% 系統成本削減 去 match 最佳靜態配置嘅準確率。關鍵 insight：full system config > LLM routing alone。

SlopCodeBench

呢個新 benchmark 嘅問題設定好正：唔係測「agent 能唔能解決一個 task」，而係測「agent 喺連續任務中點樣令 codebase 逐步退化」。

呢個係好真實嘅問題——production 入面 agent 唔係做一次就收工，係要連續維護同迭代。SlopCodeBench 將 eval 從「一次性解題」擴展到「長期代碼健康」。

🔬 評估方法論嘅三大 shift

  • 動態驗證（WANDR）：不再信靜態答案庫，實時 re-fetch 驗證
  • 系統 > 模型（Agent Arena）：full system config 比單純 model routing 重要
  • 長期 > 一次性（SlopCodeBench）：測 agent 點樣唔搞爛個 codebase

六、Physical AI：Sakana 智能細胞磚

神奇嘅係：即使冇中央控制，呢啲 cube 仍然可以推斷全局形狀、檢測損傷、再生目標結構。具體數據：

  • 六個空間方向中 95% 準確率檢測缺失鄰居

呢個係「collective intelligence from software into physical self-repairing systems」嘅具體實現。同我哋之前寫嘅 Michael I. Jordan（#22）個集體智能概念一脈相承——只係今次真係砌出實物。

八、宏觀走向：2026 年 7 月嘅 AI 格局

呢一日嘅新聞，表面係雜亂嘅「一日彙編」，但實際畫出咗幾個清晰嘅趨勢線：

1. Coding agents 大爆炸

2. 開源模型喺邊緣嘅 acceleration
Bonsai 27B（3.9GB）、Hy3 295B（單 GPU）——極端壓縮技術正在改變部署經濟學。Model 嘅質量差距在縮小，部署成本在急降。

3. 即時感知取代被動處理
MOSS-VL-Realtime 同 OmniAgent 嘅共同方向：AI 唔再係「先睇再答」，而係「一直睇住、一直思考、一直回應」。呢個 shift 對機器人、自動駕駛、監控等領域有深遠影響。

4. Eval 從「解題」轉向「持續健康」
WANDR、SlopCodeBench、Agent Arena——評估方法論喺成熟。你唔再只係問「AI 做到未？」而係問「AI 系統點樣長期健康運行？」

5. Physical Collective Intelligence 初現
Sakana 嘅智能磚係一個 prototype 級示範——冇中央控制嘅群體智慧可以 self-repair。呢個同傳統嘅「一個強大中央模型控制一切」形成對比。
