第 1 幕：AI 短劇——從小說到影片嘅 10x 壓縮

案例：STARY 副總裁 Grace

一句話：小說作者可以將自己嘅故事變成 AI 短劇，製作流程從數月縮短到數天，成本降低 90%+。

Grace 嘅公司 STARY 做嘅係 AI 短劇平台。以前小說作者要將作品影視化，要經歷：版權談判→編劇改編→選角→拍攝→後期→發行，整個鏈條從半年到兩年不等，成本動輒幾百萬甚至上千萬。

AI 短劇嘅做法唔同。作者將小說文本輸入平台，Agent 會自動進行角色提取、場景識別、對話分割，然後由多個生成模型協同工作——一個負責角色設計，一個負責場景渲染，一個負責動作編排，一個負責配音。最後由一個「剪輯 Agent」將所有素材合成為完整短片。

成個流程從數月壓縮到 數天，成本降低 90% 以上。Grace 特別提到一個細節：作者仍然係核心創作者——AI 負責執行，但創意方向同關鍵決策權喺作者手上。

「呢個模式唔係取代創作者，而係將創作者從執行層面嘅時間消耗入面解放出來，等人可以專注喺真正嘅創作上。」

行業意義

AI 短劇係「Agent 編排多模型」嘅典型場景。唔係一個模型做晒所有嘢——而係多個專用模型喺 Agent 嘅調度下協同工作。呢個模式比「一個萬能模型做晒」更實際、更可控。

另外，呢個案例揭示咗一個重要趨勢：內容生產嘅民主化。以前影視製作係高資本、高門檻嘅行業，而家一個小說作者可以單人完成。呢不僅係效率提升，更加係權力結構嘅改變——創作者直接面對受眾，中間環節被壓縮到接近零。

第 2 幕：自然語言量化——金融民主化嘅臨界點

案例：普通投資者用自然語言做量化回測

一句話：唔使寫 Code，普通投資者可以用自然語言描述策略，AI 自動生成回測代碼、執行回測、輸出結果。

量化交易以前係華爾街嘅專利——需要 Python/ C++ 功底、金融工程知識、數據基礎設施。一個普通散戶想驗證自己嘅投資想法，要麼學寫 Code，要麼俾錢請人做。

而家，你可以直接同 Agent 講：「我想試下呢個策略：買入所有 PE < 15 且 ROE > 20% 嘅 A 股，每月再平衡一次，過去 5 年嘅收益同最大回撤係幾多？」Agent 會自動生成回測代碼，從數據源拉取歷史數據，執行回測，然後用圖表+文字解釋結果。

關鍵係：Agent 唔係單純執行指令，而係會主動問問題——「你需要考慮手續費嗎？」「要排除 ST 股嗎？」「再平衡日係每月第一個交易日定固定日期？」呢啲提問逼使用戶思考自己本來唔知要思考嘅細節。

「Agent 最大嘅價值唔係『代你識』，而係幫你問出你本來唔知要問嘅問題。」

行業意義

呢個案例揭示咗 Agent 嘅獨特價值：中介能力。Agent 唔單止執行任務，仲喺任務過程中識別用戶未明確表達嘅需求，提前補充。呢種「主動中介」係傳統軟件（表單/腳本）冇辦法做到嘅。

另外，自然語言量化回測標誌住金融民主化進入新階段。唔係「人人都可以做量化」，而係「有投資想法嘅人都可以低成本驗證自己嘅想法」——呢會大幅增加市場效率，但也可能帶嚟新嘅風險（用戶對模型輸出嘅盲目信任）。

第 3 幕：藥物研發——Agent 調度嘅工作流革命

案例：AWS 藥物研發工作流

一句話：Agent 協調多個專業工具，將藥物研發中嘅分子篩選等環節大幅提速，門檻降低到「非計算化學家都可以操作」。

藥物研發傳統上係一個極度依賴專家經驗嘅領域。分子篩選——即從數百萬候選分子中找出最有潛力嗰啲——需要計算化學家用專業軟件逐個評估。一個篩選週期通常要 幾週到幾個月。

Agent 嘅做法係：定義一個工作流（Workflow），將篩選過程拆成多個步驟——分子加載→ADMET 預測（吸收/分佈/代謝/排泄/毒性）→目標蛋白對接→打分排序。每個步驟由唔同嘅專業模型完成，Agent 負責調度、傳遞中間結果、處理錯誤、生成報告。

呢個模式嘅顛覆性在於：唔係取代計算化學家，而係將計算化學家從重複勞動中解放，等人專注喺更有創造力嘅設計工作上。同樣重要嘅係，呢啲工作流而家可以由生物學家而非計算化學家操作——因為 Agent 處理咗所有技術細節。

行業意義

藥物研發案例展現咗 Agent 最有價值嘅應用場景：跨專業領域嘅工作流自動化。唔係取代專業人士，而係降低專業操作門檻，讓更多非技術背景嘅科學家可以使用計算工具。呢個模式可以擴展到材料科學、化學工程、基因組學等所有需要多工具協作嘅科學領域。

第 4 幕：遊戲開發——唔識 Code 嘅人幾分鐘生成

案例：Plaud 產品經理張庭睿

一句話：多 Agent 編排令遊戲開發門檻降低到「會講話就得」——唔識寫 Code 嘅人幾分鐘內可以生成一個可玩遊戲原型。

張庭睿展示咗一個令在場所有人都驚訝嘅 Demo：一個完全冇編程經驗嘅產品經理，用自然語言描述「我想做一個射擊遊戲，玩家控制太空船躲避隕石，每 10 秒出現一個 BOSS，打死可以升級武器」，幾分鐘之後，一個可玩嘅 HTML5 遊戲就生成咗。

背後嘅技術棧係多 Agent 編排：一個 Agent 負責理解需求同拆解任務（「呢個遊戲需要碰撞檢測、分數系統、敵人生成邏輯」），一個 Agent 負責生成代碼，一個 Agent 負責測試同調試，一個 Agent 負責 UI 同動畫。每個 Agent 都係專用嘅——類似於一個微型遊戲開發團隊。

「以前做一個遊戲原型，要揾程序員、設計師、策劃，中間溝通成本極高。而家一個人就得——唔係因為 AI 代替咗所有人，而係因為溝通成本被 AI 降到接近零。」

行業意義

遊戲原型生成係「快速迭代」哲學嘅極致體現。當一個創意到一個可交互原型嘅時間從數天壓縮到幾分鐘，創新嘅頻率同質量都會本質性提升。呢個邏輯適用於任何需要快速驗證想法嘅領域——產品設計、UI/UX、數據可視化、交互藝術。

另外，呢個案例展示咗一個重要趨勢：多 Agent 編排正在取代傳統嘅 IDE 開發流程。以前做軟件係一個人用一個工具（IDE）寫成件事。而家係一個人定義需求，多個專用 Agent 協同完成。開發者嘅角色從「寫 Code 嘅人」變成「定義問題同驗證結果嘅人」。

第 5 幕：雲廠商嘅角色轉變——從賣算力到賣生產力

以上四個案例——AI 短劇、量化回測、藥物研發、遊戲開發——背後都有一間公司做基礎設施：亞馬遜雲科技。

行業意義：雲嘅二次演化

第一次雲化（2010–2020）將 IT 基礎設施從自建機房搬到雲端。第二次雲化（2024–而家）將 AI 能力從 API 調用升級為完整嘅生產力平台。呢次演化嘅本質係：從工具到工廠——以前雲係俾工程師用嘅工具，而家雲係俾所有人用嘅生產工廠。
