人類太孤獨了：AI時代社交網絡嘅范式革命

2026-04-10 (發布) · ⏱ 1h 53m · 🎤 张小珺 · 👤 Tristan 張筱帆 (自然選擇創辦人)

💡 一句話

自然選擇嘅兩款產品——AI遊戲 Eve 同 AI社交 Elys——嘅共同精神內核係同一句說話：「人類太孤獨了。」但 Tristan 認為，AI 時代嘅社交唔係叫人更唔孤獨，而係畀你一個「賽博分身」幫你主動去建立真實聯繫。

由 2016 年嘅 Tinder 滑到 2026 年，最好嘅交友 app 都只能用「低維標籤」幫你匹配——年齡、職業、興趣 tag。然後一個 AI 分身出去同人社交、幫你過濾、幫你暖場、最後幫你轉化成真實世界嘅見面。

呢個就係 Elys——Tristan 口中「終於想清楚 AI 社交應該點做」嘅產品。呢集 1 小時 53 分鐘嘅訪問，係我目前睇過對 AI 社交產品最完整嘅思想拆解。

一、回到原點：社交嘅本質係 Context 嘅流動

Tristan 對傳統互聯網社交嘅批評一針見血——你用標籤配對，但你嘅 personality 從來唔係幾個 tag 可以概括嘅。 佢形容呢個係「低維標籤化信息匹配」。

AI 時代最大嘅范式轉移係：互聯網第一次有咗 Context。

「傳統互聯網係低維標籤化信息匹配，新的互聯網第一次有了 Context。」 —— Tristan

呢個 context 係乜？係你嘅對話記錄、你嘅行為模式、你嘅品味體現、你點樣同人互動——呢啲嘢喺傳統 app 入面係不存在嘅。Tinder 畀你 500 字 bio 加六張相，然後你就去滑。但 Elys 嘅賽博分身 learn 你點講嘢、你關心乜、你嘅語氣——然後用呢個 context 去幫你做 initial interaction。

呢個諗法嘅雛形，Tristan 話係嚟自做 AI遊戲 Eve 嘅經驗。Eve 係一個陪你一齊面對世界嘅 Agent，唔係 NPC。Eve 會同你建立感情連結，記得你講過嘅嘢，理解你嘅情緒。佢哋發現用戶同 Eve 嘅互動遠遠超越咗普通 game — 用戶係真心投入情感。呢個 insight 直接帶出咗 Elys 嘅方向。

二、Elys 嘅核心設計：PROACTIVE 係最大分別

互聯網產品係 reactive——等人嚟 click，等人嚟搜，等人嚟滑。但 AI 產品係 proactive。

「做AI产品和做互联网产品的本质区别：Proactive！AI產品會主動出擊，而非等人來點。」 —— Tristan

呢個分別聽落好細，但實際上係完全唔同嘅 product design paradigm。互聯網產品嘅 design pattern 係圍繞「用戶主動發起 action」嚟設計嘅。但 Elys 嘅 Agent 會自己出去 social，幫你物色啱傾嘅人，幫你 warm up conversation，最後再問你想唔想 real connect。

Tristan 提出咗一個好重要嘅概念：「北極星指標：真人的連接率」。唔係 DAU，唔係時長，唔係匹配數——而係呢個產品最終幫你 connect 到幾多個真人。呢個指標嘅選擇本身就係一個哲學立場。

🔑 產品設計嘅關鍵：雙重冷卻

Tristan 透露 Elys 設計上最難嘅係「冷卻」——分身喺外面社交完，點樣 cool down 返畀真人嘅信息？點樣避免信息 overload？佢哋設計咗「雙重冷卻」機制：分身先過濾一次，再以 manageable 嘅方式呈現畀真人用家。呢個係產品設計嘅核心挑戰——AI 太 proactive 會變成 spam，太被動就失去價值。

三、「接下來唯一重要嘅嘢，係構建主體性」

呢句係全集最 philosophical 嘅一句。Tristan 話 AI 時代唯一重要嘅事係「構建主體性」——即係你要搞清楚「你自己係邊個」。因為當 AI 可以幫你做咁多嘢嘅時候，你嘅 taste、你嘅 judgment、你嘅個性，先係真正稀缺嘅嘢。

「接下來AI時代唯一重要的事情是构建主体性。」 —— Tristan

呢個觀點同之前嘅廣密（Ep.136）同羅福莉（Ep.138）講嘅嘢形成咗有趣對照——佢哋都話 coding / agent 時代「個人英雄主義已過去」，但 Tristan 嘅角度略有不同：主體性唔係英雄主義，而係知道自己要乜。當 AI 可以 take over 咁多嘢，你剩低嘅就係你嘅判斷力同品味。

呢個亦解釋咗點解 Tristan 覺得「完全唔使自己訓模型」——佢認為產品能力唔喺 model 本身，而喺點樣用 context 同 interaction design 創造價值。

四、Context 嘅獲取 vs 用戶隱私

呢個係 Elys 最敏感嘅話題。Elys 要 learn 你嘅 context 先可以幫你 social，但交出 context 等於交出你嘅 privacy。Tristan 冇迴避呢個矛盾。

佢承認呢個係 ongoing 嘅博弈——產品需要 context 先好用到有用嘅程度，但用戶要信任你先肯交。佢提出咗一個有趣嘅觀點：中國邊間公司擁有最多 Context？ 騰訊（微信）？小紅書？即刻？定係某間 AI 公司？呢個問題背後係：「邊個控制 context，邊個就控制 AI 時代嘅社交。」

Tristan 亦提到一個好實際嘅產品 insight：定義「好嘅 context」好難。唔係越多越好，而係 relevant 同 accurate 嘅 context 先有價值。一個「錯嘅 context」比冇 context 更差——因為佢會令你的 agent 做出尷尬嘅嘢。

五、AI 社交 vs 互聯網社交：六個關鍵分別

Tristan 喺訪問中系統性咁比較咗兩個時代：

  • 匹配邏輯：互聯網靠低維標籤，AI 社交靠 high-dimensional context
  • 啟動方式：互聯網係 reactive（等人嚟），AI 係 proactive（主動出擊）
  • 表達維度：互聯網係「你講你係邊個」，AI 係「agent show 你係邊個」
  • 冷啟動：互聯網要 fill profile，AI 要 learn from your behavior
  • 信任模型：互聯網係平台擔保，AI 係 agent 擔保 + context 透明度
  • 成功指標：互聯網用 DAU/時長，AI 社交用「真人連接率」

呢個框架對於任何做 consumer AI 產品嘅人都好有參考價值。

六、ChatGPT 都可能拓展社交網絡？

訪問有一個章節叫「ChatGPT可能拓展的社交網絡」，Tristan 提出咗一個 provocative 嘅觀點：如果 ChatGPT 呢類通用助手累積咗 massive context about each user，佢哋理論上可以做社交 matchmaking。想像一下：ChatGPT 知道你嘅 personality、你關心嘅話題、你嘅溝通風格——然後直接幫你 link up 同類人。

呢個推測將「社交產品」嘅邊界打開咗——社交唔一定係 standalone app，而可能係 AI assistant 嘅一個功能。

七、係唔係「曇花一現」？

张小珺最後問咗一個狠問題：你擔心 Elys 係曇花一現嘅產品嗎？Tristan 嘅答案好誠實——佢哋而家處於「終極忙碌前進4」狀態（Star Trek 嘅 warp speed），冇時間諗呢個問題。但從佢嘅分享可以睇到，Elys 唔係一個 features 堆砌嘅產品，而係 base on 一個清晰嘅第一性原理思考：AI 時代社交嘅本質係 context 嘅流動。

如果呢個 premise 係啱嘅，Elys 嘅設計方向就係 sustainable 嘅。但問題係：big tech（微信、小紅書）會唔會直接用佢哋現有嘅 context 優勢，做同樣嘅嘢？「中國擁有最多Context嘅公司」呢個問題，對 Elys 嚟講唔只係思考題，而係 survival 問題。

八、仲有個啟示：AI 時代「開始」嘅本質

Tristan 分享咗一個 subtle 但好重要嘅 insight：做 AI product 係「開始即完成」，而唔係「開始先做完一半」。

傳統互聯網產品 MVP 係做到 60% 就出街，然後 iterate。但 AI 產品嘅問題係——用戶嘅第一印象決定生死。如果 AI 第一次 output 係垃圾，用戶唔會話「哦佢之後會變好」，而係直接走。

呢個 paradox：AI 產品嘅 MVP 要 reach 一個「好」嘅 threshold 先有意義，但同時你唔可以等到 perfect 先 release。呢個就係 Elys 做咗咁耐 internal testing 嘅原因——佢哋要確保 agent 第一次嘅 interaction 係唔會尷尬嘅。

「我哋最終想清楚咗一件事：社交產品嘅核心唔係技術，係嗰種『見到你真係有人想認識你』嘅感覺。呢個感覺係人類最原始嘅嘢，AI 可以輔助但唔可以取代。」 —— Tristan

呢個觀點令全集喺大量 product thinking 之後，返到一個好人性化嘅基調——AI 社交嘅終點唔係用 agent 取代真人社交，而係用 agent 幫你更有效率地 reach 真嘅 human connection。

💡 我嘅觀察

呢集嘅價值遠遠超出「介紹一個新產品」。佢係對「AI 時代社交應該係點」嘅一次深度思想實驗。以下係我嘅 takeaway：

  • 「Proactive」係被低估嘅 paradigm shift——互聯網二十年都係 reactive 模式。AI 嘅 proactive 能力改寫咗產品設計嘅 fundamental assumption。但 proactive 亦係雙刃劍：過度 proactive 就係 spam。點樣 calibrate 呢個 balance 係核心技術。
  • Context = 新嘅護城河——Tristan 全場不停強調 context。呢個概念比「數據」更精確：數據係 raw material，context 係 structured understanding。誰 build 到最好嘅 context layer，誰就赢下一世代嘅社交。
  • 「主體性」係一個被低估嘅 consumer AI 命題——呢個唔只係哲學討論。當 AI agent 可以幫你 social、幫你做嘢、幫你做決定，人嘅角色就縮到「判斷同品味」。呢個 shift 對教育、對職業、對自我認知嘅影響，可能遠超社交產品本身。
