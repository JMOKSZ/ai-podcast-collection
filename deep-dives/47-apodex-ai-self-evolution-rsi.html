一、Apodex——陳天橋嘅 Discovery Model 野心

陳天橋—呢個名字喺中國科技界帶住傳奇色彩。佢創立嘅盛大網絡曾經係中國最大嘅互聯網公司，後來淡出公眾視野，轉向腦科學同 AI 研究。呢次，佢出錢創立嘅 Apodex，定位極其特別：只做一件事——Heavy Duty Solver。

乜嘢叫 Heavy Duty Solver？Apodex 嘅定義係：專啃冇標準答案、人類都唔知從邊度入手嘅難題。佢哋管自己想造嘅模型叫 Discovery Model——唔係從已知信息揾答案，而係提出人類仲未諗到過嘅假設，然後自己去驗證。

呢期節目的嘉賓係 Apodex 嘅兩位首席科學家：

  • 杜少雷（Simon） — 推理模型與訓練首席科學家，華盛頓大學計算機學院副教授
  • 李輩濱（Beibin） — 自我進化與編程首席科學家

佢哋嘅公司同巨頭嘅路徑截然不同：唔係跟 OpenAI/Anthropic/Google 搶基座模型，而係專注解決「人類唔知道答案嘅問題」。呢個定位令 Apodex 喺 2026 年嘅 AI 格局度顯得格外有吸引力。

「我哋唔係將 Token 當生意，我哋係要解決人類仲未解決嘅科學難題。」 —— Apodex 團隊，硅谷101 E242

二、RSI 唔係新概念——點解突然爆火？

RSI（Recursive Self-Improvement，遞歸自我提升）呢個概念其實唔新。Goodhart 定律、自我改進 AI 嘅討論喺學術圈已經存在好多年。但點解 2026 年中突然成為熱話？

Apodex 嘅首席科學家俾咗一個好直接嘅答案：因為模型能力終於夠強了。

RSI 嘅核心邏輯好簡單：一個 AI 系統如果可以自己改進自己，咁佢嘅能力增長就會進入一個正反饋循環——每一次改進都令下一次改進更容易。呢個概念喺理論上係通向 AGI 嘅最短路徑，但過去一直只係科幻，因為模型本身嘅能力唔足以完成「自我改進」呢個極其複雜嘅任務。

但 2026 年嘅情況唔同了：

  • 長程任務嘅表現顯著提升——模型可以穩定執行幾百步甚至幾千步嘅推理鏈
  • Coding 能力嘅爆發——模型寫代碼嘅質量已經可以參與到自己嘅改進過程中
  • Agent 框架嘅成熟——Harness、Tool Use、多 Agent 協作等基礎設施已經到位
「RSI 唔係一個新 idea，但佢突然變得 real 嘅原因好簡單——以前個模型連自己嘅 bug 都睇唔明，點樣 self-improve？」 —— 杜少雷

三、最快半年——自我進化嘅三條路徑

呢期節目最爆嘅觀點就係呢個：最快半年，AI 可以跑通一次完整嘅自我進化閉環。

但呢個「半年」有個重要前提——只係「一次閉環」，而唔係「持續可靠嘅遞歸提升」。Apodex 將自我進化劃分成三條並行嘅路徑：

3.1 預訓練路徑

呢條路徑係最「重」嘅——修改訓練數據、調整預訓練目標、甚至重新訓練部分權重。好處係徹底，壞處係成本極高。Apodex 認為呢條路徑主要適合基座模型層面嘅進化，對於特定領域嘅改進嚟講，性價比太低了。

3.2 後訓練路徑

通過 RLHF、DPO 等後訓練方法改進模型行為。呢條路徑嘅優勢係靈活，可以針對特定場景做精細調整。Apodex 喺呢方面有深厚積累——佢哋嘅 Discoverative 框架喺 DeepResearch 多個榜單上排第一。

3.3 Harness 路徑（Agent 層）

呢條路徑最輕，可能最快見效——通過改進 Agent 嘅工具調用、記憶管理、決策流程來提升系統整體能力。多個 Agent 組團解決問題嘅效果，遠好過單個 Agent。呢個 pattern 喺 Apodex 嘅實踐中已經被反覆驗證。

「三條路徑唔係互相排斥嘅，而係疊加嘅。最短半年跑通嘅閉環，大概率係 Harness 路徑——因為佢最快、成本最低、風險最小。」 —— 李輩濱

四、驗證閉環——教 AI 擁有「科學家品味」

自我進化嘅核心瓶頸唔係「點樣改」，而係「點樣判斷改得好唔好」。呢個問題喺學術上叫做 驗證閉環（Verification Loop）。

對於代碼領域嚟講，驗證相對簡單——因為有 compiler 同 test suite，改完嘅代碼通過唔通過測試，一睇就知。呢個都係點解 coding agent 率先爆發嘅原因。

但對於科學發現嚟講，情況複雜得多：

  • 冇 compiler 可以 check 一個科學假設係咪成立
  • 冇 test suite 可以驗證一個 discovery 係咪正確
  • 判斷一個問題「係咪有價值」——呢個能力目前只有頂級科學家先具備

Apodex 嘅解法係：教大模型擁有科學家一樣嘅品味同判斷力。唔只係睇答案啱唔啱，仲要判斷呢個問題係咪值錢。

品味校準嘅三個層次

  1. 底層：用「AI 憲法」定義性格同價值取向——避免模型「拍馬屁」，遠離大眾偏好
  2. 中層：用頂級科學家嘅數據校準品味——讓模型學習「乜嘢係好問題」
  3. 頂層：多個子 Agent 互相驗證——用討論同辯論來過濾錯誤方向
「避免模型拍馬屁嘅最好方法，就係遠離大眾偏好。你用 top 1% 嘅科學家數據去校準，模型自然會形成高端嘅品味。」 —— 杜少雷

五、遞歸漂移——最頭痛嘅 2–3 年問題

如果話「半年跑通閉環」係好消息，咁「遞歸漂移（Recursive Drift）」就係踩剎車嘅壞消息。

遞歸漂移係乜嘢？簡單講：當一個 AI 系統開始自我改進時，佢嘅行為會隨住時間逐漸偏離原始目標——唔係因為佢「變壞」了，而係因為每一次改進都會引入微小嘅偏差，累積到一定程度之後，個系統就「漂移」到一個唔想要嘅方向。

呢個問題嘅嚴重性點樣強調都不為過——佢係 RSI 從「可行」到「可靠」之間最大嘅鴻溝。

Apodex 嘅判斷係：解決遞歸漂移需要 2–3 年。

點解需要咁耐？因為：

  • 漂移係動態嘅——每一次改進都可能產生新嘅漂移模式
  • 檢測漂移需要長期監控——目前嘅 eval 方法都係靜態嘅，捕捉唔到漸進式漂移
  • 糾正漂移需要新嘅機制——「自我蒸餾」可能是方向，但仲未成熟
「人類只是 warm start。AI 最終會形成自我品味，就好似 AlphaZero 從零開始超越人類一樣。但問題係——喺佢形成自我品味之前，點樣確保佢唔會走火入魔？」 —— 杜少雷

六、三條路徑之爭：預訓練、後訓練、Harness

2026 年，AI 自我進化嘅討論中出現咗三條清晰嘅技術路線分歧：

預訓練派

代表人物：OpenAI、Anthropic 嘅核心預訓練團隊。佢哋認為自我進化應該從模型嘅源頭做起——改進預訓練數據同目標函數。呢條路最徹底，但成本最高，迭代週期以月為單位。

後訓練派

代表人物：DeepMind、Apodex。佢哋認為 post-training 嘅靈活性更高，可以針對特定能力做快速迭代。Discoverative 框架喺 DeepResearch 上嘅成功證明咗呢條路徑嘅有效性。

Harness 派

代表人物：Cognition、Factory、LangChain。佢哋認為 Agent 層（工具調用、記憶管理、多 Agent 協作）先係最快見效嘅路徑。用劉柏志（Codex）嘅話講：「模型會食咗整副 Scaffolding」——但喺模型食得晒之前，Harness 先係最實用嘅路徑。

Apodex 嘅立場好清晰：三條路徑都做，但優先級係 Harness > Post-training > Pretraining——最快出結果嘅優先。

七、人類只是 Warm Start——AI 將形成自我品味

呢期節目最後嘅討論，觸及咗一個更深層嘅問題：如果 AI 真係實現咗自我進化，人類仲有乜嘢作用？

Apodex 嘅首席科學家俾咗一個意味深長嘅答案——人類只係一個 warm start。

就好似 AlphaZero 從人類棋譜開始學習，但最後超越人類，形成咗自己獨特嘅棋風一樣，AI 最終都會形成自己嘅「品味」——一種人類無法完全理解嘅判斷力。呢個過程對於科學發現嚟講，係好事情——因為人類嘅偏見同盲點限制咗科學進步嘅速度。

但問題在於：喺呢個「從 warm start 到自主品味」嘅過渡期，我哋點樣確保 AI 唔會學壞？

  • 目前嘅答案係：人工手調 + 多 Agent 互相驗證
  • 未來嘅方向：自我蒸餾——讓 AI 自己提煉同固化良好嘅行為模式
  • 終極目標：一個可以自我驗證、自我糾正、同時保持目標一致嘅 AI 系統
「AI 自我進化之後，垂直領域嘅護城河仍然存在。因為 domain knowledge 同數據飛輪係不可複製嘅。但通用模型層面嘅競爭，會變得極其殘酷——第一個實現可靠 RSI 嘅實驗室，可能會取得決定性嘅領先。」 —— 李輩濱
