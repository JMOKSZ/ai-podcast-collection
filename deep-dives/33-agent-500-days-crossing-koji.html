Agent 元年第 500 天：GUI 退場、CLI 復興、Skill 封裝——軟件嘅消亡與再生

📅 2026-07-08 ⏱ ～50 min 🎙 十字路口 Crossing 👤 主播 Koji × 嘉賓 鍾天杰（真格基金投資總監）/ 歸藏 / 戴雨森（真格管理合夥人） 🏷 Agent GUI CLI Skill Agentic Economy OpenClaw

2025 年開年，Koji 同戴雨森喺真格基金「此话当真」×「十字路口」嘅串台對談中，大膽拋出一個預測：2025 係 Agent 元年。500 天後，呢班人再次聚首——加入咗鍾天杰（真格投資總監）同歸藏（AI 開發者）——回望呢 500 日翻天覆地嘅變化。從 Claude 3.5 Sonnet 到 Manus 爆火，再到 Claude Code 刷屏、概念接連爆炸，但有一條主線逐漸浮出水面：GUI 退場、Headless 上位、CLI 復興、Skills 封裝、Agentic Economy 萌芽。

呢期唔係又一輪概念盤點，而係一次「用一條線串起所有珍珠」嘅嘗試。佢哋問嘅問題係：在 Agent 元年第 500 天，什麼在消失，什麼在誕生？

「消失的可能不是 GUI 本身，而是『人必須在流程裡當傳送帶』的前提；誕生的也不只是新產品型態，而是一整套新的分工與定價方式。」 — 節目 description

一、核心背景：500 天前，誰在喊 Agent 元年？

時光倒流到 2025 年初，LLM 熱潮經過 2023–2024 兩年嘅發酵，業界開始從「對話能力」轉向「行動能力」。Koji 同戴雨森喺當時嘅串台對談中，明確提出 2025 = Agent 元年 呢個判斷。彼時呢個論斷聽上去仲有啲大膽——Agent 概念喺學術界存在多年，但真正落地嘅產品寥寥可數。

500 天過去，回望呢段時間嘅 key milestones 可以畫出一條好清晰嘅時間線：

時間點事件意義
Day 1（2025 年初）Koji × 戴雨森對談：2025 = Agent 元年核心預測
Day ~100Claude 3.5 Sonnet 發佈；Manus 爆火Agent 嘅基礎模型成熟 + 現象級產品出圈
Day ~300Claude Code 全網刷屏編碼 Agent 成爲主流工作方式
Day ~500Concept 持續爆炸：MCP、SKill、Agentic Economy概念爆炸但主線浮現

呢 500 天嘅變化可以用一句話概括——鍾天杰喺節目入面引用咗佢自己寫嘅文章標題：

「我們也許不該再投資 GUI 思維的軟件公司。」 — 鍾天杰（真格基金投資總監）

呢個判斷唔單止係投資觀點，更加係過去 500 天 Agent 演進嘅濃縮。如果話頭 100 天仲係「AI 能對話」，第 300 天變咗「AI 能寫碼」，到第 500 天，核心問題已經變成：當 AI 唔需要 GUI 就能操作軟件，軟件本身嘅形態會變成點？

節目另一個有意思嘅觀察係：概念雖然不斷爆炸，但有一樣嘢幾乎冇變過——上下文管理越來越重要。由第一天到第五百天，呢個技術核心難題始終未變，亦係 Agent 能力嘅最關鍵瓶頸。歸藏喺節目入面嘅判斷直接了當：「上下文就是一切。」

二、主線敘事：GUI → Headless → CLI → Skills → Agentic Economy

GUI 係人類認知缺陷嘅補丁

鍾天杰提出咗一個根本性嘅觀點：GUI（圖形用戶介面）其實係人類認知缺陷嘅補丁。人類記唔到複雜嘅命令行參數，所以需要按鈕同窗口嚟降低認知門檻。但對 AI 嚟講，GUI 從來都唔係「為佢哋設計」嘅——按鈕、表單、拖拽呢啲交互方式對模型而言係冗餘嘅中介層。

「軟件沒有界面了，只剩數據庫與工具。」 — 鍾天杰

呢個觀點直接指向 Agent 時代嘅核心問題：如果 AI Agent 可以繞過 GUI，直接讀寫數據庫、調用 API、執行命令行，咁我哋仲需唔需要嗰層靚靚嘅 UI？答案係：需要，但係俾人類用，唔係俾 AI 用。

Headless 爆發——軟件嘅「斷頭」時代

Headless（無頭）軟件唔係新概念，但 Agent 嘅出現畀咗佢全新嘅生命。Notion、飛書、Google Suite 呢啲傳統 SaaS 巨頭，以前嘅 API 主要係俾開發者做集成。而家，Agent 可以通過 Codex 授權直接訪問 Gmail——唔係登入網頁版然後點擊，而係直接調用郵件 API 完成操作。

歸藏分享咗一個實際案例：佢用 Claude Code 接入自己嘅 Notion 數據庫，Agent 可以直接查詢、創建、修改頁面，完全唔需要打開 Notion 嘅 web 界面。呢種體驗喺 500 天前仲係科幻，而家已經係日常。

從真格基金嘅投資人視角睇，呢個趨勢尤其重要。鍾天杰指出：大量公司主動推出 MCP（Model Context Protocol）/CLI，唔係被逼嘅，而係意識到「如果唔開放，Agent 時代就冇你份」。呢個觀察喺投資層面嘅含義係：

  • 下一代軟件公司應該「API-first」甚至「Agent-first」
  • 傳統 SaaS 公司需要盡快開放 Headless 接入
  • 封閉生態系統面臨被 Agent 繞過嘅風險

CLI 復興——AI 拯救咗命令列

節目入面嗰個最反直覺嘅觀察係：CLI（命令行介面）正在復興。點解一個喺 GUI 時代被認為「過時」嘅交互方式會翻生？答案好簡單：AI 解決咗 CLI 最大嘅障礙——人類根本記唔到成千上萬條命令。

以前，命令列係專家工具，因為你要記住 `grep`、`awk`、`sed`、`curl` 嘅複雜參數。而家，你只需要用自然語言話俾 Agent 聽你想做乜，Agent 自動生成命令並執行。CLI 嘅純文本本質令佢天然適合 AI 處理——結構化、無歧義、可組合。

「CLI 的核心：命令行執行、純文本、與 AI 天然契合。」 — 節目關鍵詞拆解

壟斷性軟件嘅防守困境

節目嘅討論去到一個極具爭議嘅話題：微信、小紅書、美團等壟斷性軟件至今沒有推出 CLI。呢唔係技術問題，而係戰略問題。開放 CLI 意味住第三方 Agent 可以直接操作呢啲平台嘅核心功能——發朋友圈、落單、落廣告——呢啲功能一旦開放，平台對用戶行為嘅控制力就會減弱。

但另一方面，如果呢啲平台唔開放，Agent 時代嘅用戶自然會流向開放生態。呢個係一個「開放 CLI 防守兩難」：

  • 開咗： 失去對生態嘅絕對控制，可能被 Agent 供應商抽成
  • 唔開： 被 Agent 用戶拋棄，淪為「人類專用」嘅遺留系統
「開放 CLI 背後需要極大決心與勇氣。」 — 節目中對壟斷軟件嘅點評

Koji 仲提出咗一個有趣嘅反問：如果工具能通過 Agent 更好地找到用戶，咁開放 CLI 其實係「用入口換生態」。呢個判斷喺節目入面引發咗激烈討論。歸藏舉咗一個例子：年輕人已經開始喺抖音上聊天，DAU 1 個億——呢啲平台本身就係新時代嘅「操作系統」，佢哋點可能輕易開放核心功能？

「作為 PM，最重要是定義問題」

「作為 PM 最重要的是定義問題，不要去解上個時代已解決的問題。」 — 節目金句

呢句說話指向咗更深層嘅問題：喺 Agent 時代做產品，唔應該諗「用戶點樣喺 GUI 入面操作」，而係「用戶最終想要嘅結果係咩」。如果結果係「發一封郵件俾客戶」——Agent 可以直接做，唔需要用戶打開 Gmail->點「撰寫」->輸入內容->點「發送」。呢個思維轉變，可能係 500 天後最值錢嘅 insight。

三、Skill 封裝——你的品味下沉到 AI

歸藏嘅 PPT Skill 案例

節目入面最生動嘅案例嚟自歸藏。佢展示咗點樣用 Claude Code 做一個「PPT Skill」——佢唔係教 Claude 做一個好嘅 PPT，而係將自己對「好 PPT」嘅品味提煉成可複用嘅指令集合。呢個過程唔係寫 prompt 咁簡單，而係一個反覆「拷問」嘅過程：

「讓 Claude Code 拷問你的 grill-me——不斷追問你『呢個設計點解好？呢個段落點解要咁寫？』——最終提煉出你自己嘅品味標準。」 — 歸藏

呢個方法論嘅核心係：Skill 唔係教 AI 新嘢，而係將你嘅品味沉澱落去。每一個 Skill 文件，本質上都係嗰個人在某一領域嘅「品味容器」——入面封裝咗佢嘅判斷標準、風格偏好、質量門檻。

Skill 會自我沉澱——品味嘅 moat

呢度觸及咗一個更深層嘅 insight：Skill 嘅真正價值唔係「功能」，而係「品味」。當你一次次迭代你嘅 Skill 文件，佢唔單止係一個 prompt template，而係你作為一個專業人士嘅判斷力外部化。歸藏嘅 PPT Skill 唔單止令 Claude 出到好嘅 PPT，而係令「歸藏風格」嘅好 PPT 可以被複製同迭代。

「Skill 會自我沉澱，把你的品味提煉成可復用的結果。」 — 歸藏

呢個諗法極具前瞻性：當你的 Skill 文件積累到一定規模，佢就變成咗你嘅「數字延伸」——一個喺 AI 世界入面代表你品味同能力嘅代理資產。呢啲資產嘅累積效應比傳統嘅作品集更強大，因為佢哋係「可執行嘅知識」。

共存還是過渡？

節目對 Skill 嘅長期命運進行咗有意思嘅討論：Skill 係長期存在定係過渡形態？歸藏認為，隨住模型能力嘅不斷增強，一部分簡單嘅 Skill 最終會被模型本身消化（模型本身就「知」點樣做好 PPT），但嗰啲高度個人化、承載專業品味嘅複雜 Skill 會長期有價值。

鍾天杰從投資人角度補充：小紅書 Skill 商店 如果做得好，可能成為一個新嘅分發平台。創作者唔單止可以賣內容、賣課程，仲可以賣「品味封裝」——即係一組讓 AI 幫你做好某件事嘅 Skill。呢個商業模式而家仲處於極早期，但從平台角度睇，佢嘅想像空間好大。

三位嘉賓仲釐清咗一個重要區分：

概念定位適用場景
CLI低層次工具調用協議系統級操作、文件處理、環境配置
MCP（Model Context Protocol）標準化上下文協議Model 與外部工具嘅標準化互動
Skill品味/專業知識封裝高質量輸出、個人化風格、專業判斷

三者唔係競爭關係，而係唔同抽象層次嘅產物。CLI 係執行層，MCP 係協議層，Skill 係知識層。

四、Agentic Economy（智能體經濟）萌芽

如果話前三部分講緊「技術形態點樣變」，呢部分講嘅係「成個經濟系統點樣重構」。節目提出咗一個大胆嘅斷言：Agentic Economy 嘅基建時代已經開始。

To Agent 基礎設施

當 Agent 成為經濟活動嘅主體，成個基礎設施層需要重做：

  • 沙箱（Sandbox）： Agent 執行需要安全隔離嘅運行環境，確保佢哋唔會意外/惡意影響宿主系統
  • 記憶系統（Memory）： Agent 需要持久化記憶，跨越 session 嘅上下文管理——呢個就係歸藏講嘅「上下文就是一切」
  • 支付（Payment）： Agent 之間、Agent 與人類之間點樣結算？呢啲仲未有成熟方案
「現在仍是大基建時代，Token 變得越來越智能、越來越便宜是主旋律。」 — 鍾天杰

Token 經濟學

節目花咗唔少篇幅討論 Token 嘅經濟屬性。傳統上我哋將 Token 視為「計量單位」——好似電力度數咁。但鍾天杰指出，Token 嘅價格其實會浮動，受多個因素影響：

  • 能源成本： 訓練同推理嘅電費佔比會越來越高
  • 算力供給： GPU/NPU 芯片嘅供應鏈瓶頸
  • 模型水平： 更聰明嘅模型可以用更少 Token 完成更多工作（變相降低單位成本）
  • 市場供需： Agent 數量爆炸會推高 Token 需求

呢個分析嘅底層邏輯係：Token 唔單止係技術概念，佢越來越似一種「計算貨幣」——有供需曲線、有價格彈性、有替代品。當 Agent 經濟規模達到一定程度，Token 嘅估值模型可能獨立於傳統貨幣體系。

五、OpenClaw 嘅遺產

呢期節目特別花咗一節講 OpenClaw——一個由 Koji 團隊主導嘅開源項目。呢個項目喺 Day ~100 前後爆火，但到第 500 日，佢嘅熱度已經極速消退。佢哋坦誠地討論咗呢個現象，同時亦梳理咗 OpenClaw 留低咗嘅遺產：

100 天熱度極速消退

OpenClaw 嘅故事係一個典型嘅開源 Agent 項目生命週期：爆紅→迅速增長→遭遇天花板→熱度消退。呢個軌跡唔係失敗嘅標誌，而係揭示咗一個重要嘅規律：開源 Agent 框架嘅「新鮮感窗口」好短。新項目一出嚟大家爭住試，但一旦跨過「好奇期」進入「實際使用期」，用戶會好快評估佢嘅真實價值。

塑造共識：最大嘅遺產

「OpenClaw 最大意義是塑造共識，幫大眾完成對 Agent 的認知轉變。」 — 鍾天杰

呢個判斷好到位。OpenClaw 嘅產品意義可能唔及佢嘅「文化意義」——佢係第一個讓大量非技術用戶實際體驗「AI Agent 係乜嘢」嘅開源項目。喺 OpenClaw 之前，Agent 係一個抽象概念；喺 OpenClaw 之後，成千上萬嘅人親手用過一個「自己運行嘅 Agent」。

呢種「認知轉變」嘅作用唔好低估。每個技術浪潮都需要一個「入門毒品」——一個低門檻、可體驗嘅產品，幫普羅大眾越過概念門檻。OpenClaw 就係 Agent 浪潮入面嘅那個角色。

IM 形態證明

另一個被低估嘅遺產係：OpenClaw 證明咗 IM（即時通訊）形態嘅價值。用戶唔需要一個新嘅 IDE、一個新嘅 Dashboard——佢哋想要嘅係喺熟悉嘅界面（微信/Telegram/Discord）入面同 AI 對話。呢個觀察直接指導咗後續好多 Agent 產品嘅設計方向。

「用戶需要在熟悉界面裡與 AI 對話。」 — 節目中對 OpenClaw 遺產嘅總結

從呢個角度睇，OpenClaw 嘅真正遺產唔係代碼庫，而係：一套關於「人機交互應該係點」嘅設計共識，同一個讓 Agent 平民化嘅認知轉折點。

六、Token Grant：「缺的不是錢，是 Token」

節目尾聲介紹咗一個由真格基金同十字路口聯合發起嘅計劃——Token Grant。呢個計劃嘅概念好直接：每個入選項目獲得 5 萬元 Token 費用，幫創業者解決「有想法但冇算力」嘅問題。

Code Pilot：6000 GitHub Stars

其中一個案例係 Code Pilot，一個已經有 6000 GitHub stars 嘅開源項目。佢嘅定位係「足夠開放、本地化嘅 Agent」，強調用戶對自己 Agent 嘅完全控制權。呢個項目嘅增長說明咗一件事：市場對「本地優先」、「隱私可控」嘅 Agent 方案有真實需求。

0 行代碼到 10 萬行代碼——一場數字生命實驗

節目仲提到一個引人入勝嘅故事：一場數字生命實驗——一個項目從 0 行代碼 開始，全程使用 Agent 編程，成長到 10 萬行代碼。呢個案例本身唔單止係技術 demo，而係一個深刻嘅里程碑：Agent 已經可以完整支撐一個真實項目嘅全生命週期開發。

「今天創業缺的不是錢，而是 Token。」 — 節目中 Token Grant 嘅核心觀點

呢個 statement 反映咗一個時代變化：喺傳統 startup 生態入面，錢係最稀缺嘅資源；喺 Agent 時代，Token（即係算力 + 模型能力）取代咗錢嘅位置。當你嘅「員工」係 AI Agent，佢哋唔需要薪水，但需要 Token。呢個轉變對創業融資模式嘅影響，可能喺未來幾年逐漸浮現。

八、我的洞察

聽完呢期對談，有幾個觀察特別值得拎出嚟獨立思考。

GUI → Headless 係認知革命，唔係技術革命
