私有 Eval 即 IP——Satya Nadella 的 AI 平台戰略與 Harness 時代宣言

📅 2026-06-04 ⏱ 約 40 分鐘 🎙 Sarah Guo・Elad Gil・swyx 👤 Satya Nadella（Microsoft CEO）
私有Eval 多模型Harness AI平台戰略 MAI模型 SaaS終局

Microsoft Build 2026 嘅 Keynote 之後，Satya Nadella 冇去休息——佢直接去咗一個 crossover podcast session，由 No Priors（Sarah Guo + Elad Gil）同 Latent Space（swyx）一齊圍攻。

呢 40 分鐘唔係一個 CEO 嘅公關訪問。Satya 嚟咗真——佢講清楚咗 Microsoft 喺 AI 時代嘅第三條路：唔係 OpenAI 嘅模型第一，唔係 Google 嘅全棧自研，而係 開源 Harness + 私有 Eval = 企業 AI 平台。

如果你只可以聽一個科技 CEO 喺 2026 年對 AI 嘅戰略判斷，好大機會係呢個。

全場最核心一句：私有 Eval 係公司最大嘅 IP

呢個唔係修辭，係 Satya 嘅戰略核心。

「每間公司擁有私有評估（private evals）可能係最大嘅 IP。諗下——你有一個私有 eval，你可以用一個前沿模型去 hill climb，而且唔泄漏 traces。呢個可能係未來知識產權最大嘅驅動力。」——Satya Nadella

佢仲提出咗一個 acid test：

「你有一個私有 eval。你用緊 Model A。你可唔可以 switch 去 Model B，再 climb up？如果可以——你 control。如果唔可以——你唔 control。呢個就係點解 Harness 嘅決策變得咁重要。」——Satya Nadella

呢個測試重新定義咗「企業 AI 自主權」。唔係你有冇自己嘅模型，而係你可唔可以自由切換模型而唔損失你嘅評估能力。

Bill Gates Line：生態系 vs 單一平台

Sarah Guo 問咗全場最好嘅問題之一：Microsoft 嘅角色係平台定係模型公司？

Satya 用咗佢喺 Microsoft 經歷四個平台時代嘅視角嚟回答——由 Windows 到雲端到 AI，不變嘅一條原則：

「一個平台嘅定義，係佢有能力創造更多價值 圍繞住 個平台，而唔係 被 個平台 capture 晒。」——Satya Nadella

呢個就係所謂嘅「Bill Gates Line」——Microsoft 嘅生態系統必須令客戶得到嘅價值遠超過 Microsoft 自己 capture 嘅價值。如果唔係，就唔會有開發者喺你上面起嘢。

Satya 仲用咗一個有趣嘅自嘲：「我成日同 Jensen 講——你 build CUDA 喺我哋嘅 DirectX 上面，God，我 got the short end of that。」

MAI 模型嘅真正策略：乾淨血統 + Hill Climb Scaffold

Microsoft 宣佈咗 MAI 系列模型——由細到 5B 到大型 frontier 模型。但重點唔係模型本身，而係佢哋點樣 train。

  • 乾淨血統（Clean Lineage）：訓練數據嘅質量同消融實驗變得比任何 benchmark 更重要。Satya 話「好多 open weight 模型一兩個 benchmark 好靚，但實戰唔得」——呢個唔單止係批評，係佢哋 design principle 嘅反面教材
  • Hill Climb Scaffold：模型只係起點。你要圍繞住模型建立「hill climbing scaffold」——一個可以幫助模型喺特定領域爬得更高嘅環境
  • Private Evals + Traces = RLE：你開始收集 traces，建立自己嘅 reinforcement learning environment（RLE）。呢個 loop 就係每個公司嘅競爭壁壘
  • 時間性（Temporality）成為新 frontier：Satya 用咗一個例子——你可以先用一個強模型（GPT-55）生成一堆 traces，然後用嚟 train 一個 5B reasoning model，令佢喺特定任務上超越大模型
💡 關鍵 Insight：模型正在從「產品」變成「原材料」。真正嘅產品係你點樣 hill climb——你嘅 eval、你嘅 trace、你嘅 scaffold。呢個就係點解 Satya 話「私有 Eval 係新 IP」。

代碼太成功了，AI 需要重新定義 IDE

一個好有趣嘅 paradox。Satya 話：

「Coding 太成功，以至於我哋而家需要 rebuild 個 IDE。你有 100 個 agent sessions 同時運行——佢哋交返嚟嘅 cognitive load 作為人類根本搞唔掂。Chat 作為唯一嘅 artifact 都係唔可能嘅——所以我哋需要 Canvas。」——Satya Nadella

呢個 paradox 好深刻：AI 越幫到你，你越需要新嘅 UI 嚟同 AI 協作。傳統 IDE 係俾一個人寫 code 嘅——而家係俾一個人管理 100 個 AI 寫 code。介面要完全反思。

Satya 仲講到「glue work」——人類大量時間花嘅係連接唔同系統嘅膠水工作。而家可以用 long-running、durable 嘅 agents 嚟做呢啲 glue work。「六個月後，我哋所有人都會話——嘩，我一晚之間有咁多 autopilots 喺度幫我做嘢。」

SaaS 嘅終局：Build vs Buy 方程已變

Elad Gil 問咗一個好敏感嘅問題：SaaS 模式會唔會死？既然任何 workflow 都可以 cheaply generate，咁仲有咩需要用買嘅？

Satya 嘅答案好有層次。佢話傳統 SaaS 嘅做法係：我哋揀一個 business process → 建立 data model → 寫 business logic → 加上 UI → 少少 configuration。呢個模式做咗 20 年。

而家發生嘅係：得 AI 之後，你唔需要買一個完整嘅 SaaS 產品去 capture 一個 workflow——你可以 prompt 一個 agent 去做。但呢個唔代表 SaaS 會死。Satya 嘅觀點係：終局係一個混合體。

「每間公司都會有自己嘅 agents 做緊工，人類做緊工，兩者之間嘅 traces——呢啲先係企業點樣創造價值嘅真正 context。呢啲 traces 返去 train 嘅唔係一個 generalist model，而係 公司老兵 agent（company veteran agent）。呢啲嘢要上 balance sheet 㗎。」——Satya Nadella

換句話講：SaaS 唔係死——SaaS 係變成 Agent-as-a-Service。以前你買一個 CRM 軟件包管理客戶關係，第時你可能買一個「銷售 agent」——佢帶住佢自己嘅 traces 同經驗嚟幫你。

💡 我嘅 Insight

呢集最好嘅嘢唔係 Satya 講咗咩 specific 產品，而係佢點樣重新定義咗 Microsoft 嘅 AI 戰略框架。

第一，「私有 Eval 係新 IP」呢個概念會改變好多嘢——如果評估能力比模型本身更重要，咁投資者評估 AI 公司嘅方式、企業採購 AI 嘅方式、甚至係學術界發表研究嘅方式，都要重新調整。

第二，Satya 對「平台 vs 模型」嘅取態其實係一個幾高明嘅 positioning——當全世界都喺度拗「邊個模型最好」嘅時候，Microsoft 話「我唔 care——你鍾意用邊個都可以，用我嘅 harness 同 eval infrastructure 就得」。呢個係典型 Microsoft 打法：product 層面唔一定要贏，platform 層面贏就得。

第三，「Can everybody operate at the frontier with their frontier intelligence？」呢個 tagline 背後係一個好大嘅假設——每間公司都有能力建立自己嘅 frontier eval。真係？對於大型企業，可以。對於 startup？可能佢哋需要另一個平台幫佢哋做呢件事。

最後——Satya 話「swyx，你知唔知你講緊嘅係 SEC 需要為 token expertise 設立會計標準」嘅時候，佢係認真嘅。呢個唔係講笑。如果 AI agents 嘅 traces 變成可以上 balance sheet 嘅資產，成個企業估值體系都要改。

重點引用

「如果冇呢個平台承諾——你嚟 build with us，你會得到屬於你 data 嘅 intelligence——我唔知點樣達到穩定均衡。冇咗呢個，直接 worship at the altar of one model 就得，使乜開 developer conference？」——Satya Nadella
「人類資本從未可以被放上 balance sheet——因為你冇辦法捕捉 tacit knowledge。而家你可以——透過 agent 隨時間學習嘅 traces。」——Satya Nadella
「你可以用 Llama harness，可以用任何 open harness，或者你自己嘅 harness——然後用你嘅 tools、multiple models、你嘅 context 嚟 train。呢個先係我哋嘅 pitch。」——Satya Nadella
