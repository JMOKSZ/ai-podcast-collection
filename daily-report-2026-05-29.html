🎙️ The Age of Async Agents — Cognition's Walden Yan & OpenInspect's Cole Murray
Latent Space (swyx + Alessio) · 2026-05-28 18:41 UTC · 约 68 分钟
Show Notes · YouTube
🎯 主题

Cognition 联合创始人兼 CPO Walden Yan 与 OpenInspect 创始人 Cole Murray 深度探讨背景 Agent（Background/Cloud Agents）的崛起——从本地 IDE 辅助到云端自主编程 Agent 的历史性转折。重点分析 2025年12月模型拐点如何催生了「Spec-to-PR」生产流程，以及 Cognition 的 Devin 如何在数月内将内部提交占比从 16% 推至 80%。

👤 嘉宾背景
  • Walden Yan — Cognition 联合创始人 / CPO。「上下文工程（Context Engineering）」概念提出者。2025年发表《Don't Build Multi-Agents》，2026年4月转变立场，公开承认找到有效多 Agent 方案。
  • Cole Murray — OpenInspect 创始人。开源背景 Agent 系统构建者，为企业提供可定制云端编码 Agent 基础设施。
💡 核心论点
  • 三波 Agent 进化论：
    • 第一波 — Copilot/补全：开发者全程在场，IDE内逐补全接受/拒绝，由开发者本地工作流串行瓶颈。
    • 第二波 — 本地 Agent：Claude Code、Windsurf、Cursor Agent 面板——多终端并发运行，但仍锚定在开发者本地环境。
    • 第三波 — 异步/背景 Agent：Agent 不依附于本地 IDE；任务、仓库、沙箱、shell、浏览器、测试、Memory 均在云端，通过 GitHub bot / Slack 集成调用，实现真正的端到端软件开发。
  • 2025年12月模型拐点：以 Opus 4.5 和 GPT 5.2 为代表的模型达到「从规范到完整 PR」的能力水平。Cole Murray 指出此拐点彻底改变了人机交互模式——人不再需要手把手引导模型，可以给出足够好的 spec 后放手让 Agent 独立完成。
  • 心态逆转：2025年6月 Walden 发文《Don't Build Multi-Agents》，强调上下文工程而非多 Agent 架构；到2026年4月，他已公开承认「很多看似性感的想法仍然不实用，但我们找到了一些真正有效的架构」。
  • Cursor 范式转变：引用 Cursor 联合创始人 Michael Truell 的《The Third Era》——「Cursor 不再主要是关于写代码，而是帮助开发者构建制造软件的工厂。这个工厂由 Agent 舰队组成，开发者以队友身份交互：给出方向、配备工具、审查成果。」
📊 关键数据
  • Cognition 刚完成 $1B Series D，估值 $260亿，由 Lux Capital、General Catalyst、8VC 领投
  • 企业使用量年初至今增长 10x+；年化经常性收入（ARR）达到 $4.92亿
  • Devin 在 Cognition 内部仓库的提交占比从 2026年1月的 16% 飙升至 3月的 80%
  • Merged PR 数量在2-3个月内增长 7x
  • 同期工程团队人数仅增长约 10%——Agent 产出与人力增长极度脱钩
  • Agent 预算范围：$1,000–$5,000/工程师
📌 重点引用
「我们看到了一个转变，时间点大约是2025年12月。Opus 4.5 和 GPT 5.2 达到了一个能力阈值，我们不再需要手把手引导模型——只要 spec 足够好，就可以几乎无摩擦地完成从规范到完整 PR 的流程。」—— Cole Murray
「Cursor 不再主要是关于写代码。它是关于帮助开发者构建制造软件的工厂。这个工厂由 Agent 舰队组成，开发者以队友身份交互。」—— Michael Truell / Cursor
「我们一直认为它在加速，但今天回头看过去三到四个月，加速甚至更快了。Sonnet 3.7 在当时看起来是一个巨大飞跃，但实际上很多工作只是去掉了 Devin 中不再需要的部分。」—— Walden Yan
🔧 技术细节
  • Devin 架构：「大脑」与「机器」分离。 大脑（Agent 逻辑运行处）与执行环境（全 VM / 沙箱）解耦，实现安全隔离和权限精细化。
  • 全 VM 方案： 不仅 Docker——Agent 需要运行真实应用、测试 Android、macOS、Windows 环境，需要嵌套虚拟化。Docker 不够——因为一些真实应用和端到端测试不能在 Docker 容器内运行。
  • Harness In vs Out of the Box： Agent 执行环境的选择——GPU、存储、计算资源是否打包在一起。
  • 视频验证： Agent 完成后生成测试视频片段，人工审阅确认「我知道它工作」，然后合并。
  • 自动记忆生成： Devin 自动构建项目记忆，但 Memmory pruning（记忆修剪）仍是未解决的难题。
  • MCP 不足： MCP 协议不足以支撑企业级 Slack 集成等一等公民集成需求。需要更深层集成。
  • 多 Agent 编排： 子 Agent、Meta-Devin 管理，真正的多 Agent 系统架构正在形成。
🧠 行业 Insight
  • 「每个人都建自己的 Devin」已成趋势： 从 Shopify、Stripe、Ramp 到 Paradigm、Razorpay，各家都在自建背景 Agent。甚至 Cognition 的「朋友们」也在自建。但 Cognition 并不感到威胁——他们超募。
  • $20/seat 的 Agent 产品是「死亡区」： Cole Murray 明确表示不想卖 $20/seat——太容易被复制，利润在沙箱层（Daytona、E2b）和模型层，中间层很难货币化。
  • 纯 Vibe Coding 的失败模式： 无管控的自动合并 Vibe Coding 大约「两周后就会崩溃」。代码库会退化到最差工程师的水平——AI code smells、reward hacking 等问题。
  • Windsurf 2.0 的定位： 本地前台 Agent 与云端后台 Agent 之间的分工和交接——这可能是 IDE 的未来形态。
  • SRE 自动分诊已成现实： Agent 可作为第一响应人处理工单和事故。
  • 非工程师也在用 Agent： PM、市场人员已可通过 Slack 直接创建 PR。
🔄 前後脈絡

本期是 Latent Space 对 Agent 系列深度的延续。此前已有 Claude Code、Steve Yegge 的 Vibe Coding Manifesto、Windsurf 等多期。Walden 本人的观点演变（2025年6月「别建多 Agent」→ 2026年4月「找到有效方案」）本身就是最佳叙事——代表了整个行业对 Agent 认知的快速成熟。

🎙️ Building an AI Guardian for Enterprise with Onyx Security CEO Maxim Bar Kogan
No Priors (Sarah Guo + Elad Gil) · 2026-05-28 10:00 UTC · 约 42 分钟
YouTube
🎯 主题

随着 AI Agent 的能力接近可实际运行电网和水资源管理，如何防止它们「失控」成为紧迫问题。Onyx Security 创始人兼 CEO Maxim Bar Kogan 与 Sarah Guo 深入对话，探讨企业级 AI Agent 的监控、安全和治理架构。

👤 嘉宾背景
  • Maxim Bar Kogan — Onyx Securities (Onyx) 创始人兼 CEO。前以色列国防军技术背景，公司聚焦于 AI Agent 安全控制面板（AI Control Plane）。自称「全情投入 AGI」的人。
  • 公司扎根以色列，利用该国强大的 AI 和安全人才生态。
💡 核心论点
  • Agent 安全需要「AI 控制面」而非传统防火墙： Onyx 的产品本质上是一个 AI 控制面板，监管 Agent 的权限和灵活上下文。它监控 Agent 在「做什么、为何做、环境是否合适」，而不是传统的网络/端点安全。
  • Proxy 模式不适用于 AI Agent： 传统安全 proxy 拦截流量进行审计，但 Agent 的行为复杂多变，需要实时理解意图和上下文——这超出了 proxy 的能力。
  • Onyx 自己训练模型做安全： 现成的 LLM 没有足够的安全上下文（context）来理解 Agent 的意图。Onyx 训练自己的安全模型，专门用于监控 Agent 行为。
  • 逐步模型部署策略： 他们采用分阶段部署框架「Glasswing」（玻璃翼）和「Daybreak」（破晓），逐步将更强大的模型推向生产。在不同阶段评估安全风险。
  • 需要供应商无关的监管层： 企业不应该只依赖模型提供商（OpenAI/Anthropic/Google）自身的安全声明。需要一个独立的、跨模型的监管层。
  • 可解释性（Mechanistic Interpretability）是关键： 只有理解模型内部机制，才能真正保证安全。
📌 重点引用
「我们比以往任何时候都更接近这样一个世界：AI Agent 足够聪明，可以运行我们的电网和管理水资源。那么问题来了——我们如何防止它们失控？」—— Sarah Guo（开篇引言）
🔧 技术细节
  • Onyx 产品 = AI 控制面板： 可监管 Agent 的权限、上下文、执行环境。需要平衡延迟、成本和可靠性。
  • 自我训练的安全模型： 因为通用模型缺乏 Agent 安全上下文的深度，Onyx 专门训练安全分类/监控模型。
  • Glasswing 和 Daybreak： 两个命名框架，分别用于不同安全层级的 Agent 逐步部署。
  • 机制可解释性： 他们投资于理解模型内部运作，以从根本上保障安全。
🧠 行业 Insight
  • 企业还没准备好： 大型企业对 Agent 安全普遍认识不足。"剩下来的持观望态度的企业"是最大的市场增长空间。
  • 以色列的 AI 安全生态正在崛起： Maxim 强调以色列的安全和创新人才密度是公司核心竞争力。
  • 对 AGI 的「all-in」态度： Maxim 明确表示他是「AGI-pilled」——即坚信 AGI 的到来并将公司定位为 AGI 时代的安全基础设施。
  • 实验室的责任边界： 讨论延伸到模型实验室（OpenAI、Anthropic 等）应承担多少安全责任。Maxim 认为实验室确实需要改进模型信任和治理，但企业不能仅依赖实验室层安全。
🔄 前後脈絡

本期是 No Priors 安全系列的延续。此前有 Baseten CEO Tuhin（推理云）、Cerebras CEO Andrew Feldman（芯片IPO）、Pax Silica 的 Jacob Helberg（地缘供应链）——话题从底层基础设施、芯片、地缘政治，现在上升到 AI Agent 安全治理，显示 AI 行业纵深正在打开新的安全赛道。

🎙️ Reiner Pope – Chip design from the bottom up
Dwarkesh Podcast (Dwarkesh Patel) · 2026-05-22 15:38 UTC · ~90 分钟
Show Notes · YouTube
（上轮已收录，本次更新补充完整精华萃取）
🎯 主题

Reiner Pope（MatX CEO，前Google TPU架构师）在黑板上从零开始讲解 AI 芯片工作原理：从基础逻辑门出发，逐步构建到 GPU、TPU、FPGA、人脑的各自设计哲学。纯粹的技术性内容，适合硬件从业者和深度技术爱好者。

💡 核心论点
  • 乘累加（Multiply-Accumulate）是 AI 芯片的原始原子操作： 矩阵乘法的一切都归结为 MAC。每个矩阵乘法步骤 = output[i,k] += input[i,j] × other[j,k]。
  • 精度非对称性： AI 芯片中乘法精度通常低于累加精度——因为乘法只有一次误差，而累加是多次求和，误差累积快。举例：4-bit 乘法 × 8-bit 加法。
  • 芯片面积随位宽二次方增长： 这就是为什么低精度运算在神经网络中如此有效。FP4 面积不是 FP8 的一半，而是 1/4 或更少。
  • Nvidia 已经开始承认精度缩放不是线性： 历史上每次精度减半（FP8→FP4），FLOP 翻倍。但 B300 及之后，FP4 是 FP8 的 3x 而非 2x。Reiner 指出考虑到二次缩放，理论上应该是 4x——但浮点格式的指数位带来额外开销。
  • GPU 本质是很多很小的 TPU： GPU 核心与 CPU 核心相比小而多，就是一组微型的专用计算单元。
  • Mux（多路选择器）的成本被软件工程师严重低估： 一个看似简单的「选择第三个寄存器」操作，在硬件层面需要 n×p 个 AND 门和 (n-1)×p 个 OR 门。数据搬运的成本远超运算本身。
📊 关键数据/事实
  • Dadda 乘法器：p-bit × q-bit 整数乘法需要 p×q 个全加器。输入：p×q + p+q 位；输出：p+q 位。
  • 4-bit × 4-bit 乘累加 = 16 个 AND 门（生成部分积）+ 16 个全加器（压缩求和）。
  • Nvidia B300 FP4 性能 = FP8 的 3x（理论上应接近 4x，因浮点指数位开销）。
  • CPU Core 面积远大于 GPU Core——因为前者需要复杂控制逻辑（分支预测、乱序执行），后者几乎全是 ALU。
📌 重点引用
「芯片设计中的一个主要选择是：我放多少 FP4 逻辑，放多少 FP8 逻辑？有时从客户需求考虑，有时为了均衡 FP4 和 FP8 的功耗预算。」—— Reiner Pope
「每次精度减半，性能翻倍——这个比率其实不太对。因为面积是二次方缩放的，你应该得到比预期更大的加速。Nvidia 在 B300 及以后的产品规格中也开始承认这一点。」—— Reiner Pope
「GPU 就是一群很小的 TPU。」—— Reiner Pope
🔧 技术细节
  • 全加器 = 3→2 压缩器： 三个单 bit 输入 = 两个 bit 输出（和 + 进位）。本质是「数数然后转二进制」。
  • Systolic Array（脉动阵列）： 数据像心跳一样在芯片上流动，每个计算单元只做非常简单的事，最大化数据复用，最小化内存访问。
  • FPGA vs ASIC： FPGA 可重配置但慢且功耗高；ASIC 固定但快且省电。AI 训练推广大规模使用 ASIC（GPU/TPU）。
  • Cache vs Scratchpad： Cache 是硬件自动管理，Scratchpad 需软件显式管理。后者在确定性负载（如矩阵乘法）中更优。
  • 时钟周期和流水线寄存器： 逻辑门之间通过流水线寄存器同步，决定芯片能跑多快的主频。
🔍 行业 Insight
  • 低精度推理的市场核心： 几乎所有 AI 推理都用低精度（FP4/FP8/INT8），这是 Nvidia 和创业公司竞争的焦点。MatX 在此战场设计差异化芯片。
  • Crusoe 的差异化： 广告中提到 Crusoe 是少数达到 SemiAnalysis ClusterMAX 黄金级的 GPU 云，TCO 比白银级低 5-15%。早期故障检测和快速节点替换是隐性但关键的优势。
  • Jane Street 的 AI 态度： Dwarkesh 与 Jane Street 的对话显示这家低调的量化巨头在积极训练自己的模型，比以往更重视招募技术人才。
